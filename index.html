Based on HTML

Build your Web applictions quickly and efficiently by using HTML syntax.

Implemented with jQuery

jTags is implemented with jQuery providing support for all major browsers.

The missing piece

jTags reduces server load by placing redundant operations in the hands of the client.


jTags are a unique way of building a highly dynamic, customizable and efficient website quickly and easily. With jTags you can embed functionality to the markup tags to make them perform a certain task.

The idea behind jTags

The standard jTags that come pre-built with the core library are ment to provide an easy way to create a web application that runs mostly on the client. This change in app methodology makes the website much more efficient, faster, and much more dynamic due to the reduced laod on the server(s).

The problem:

Currently, servers provide significant ammounts of redundant data instead of efficiently providing only the data that is needed. This is because the Web was designed for static html pages as apposed to dynamic Web applicaions.

For instance, every time you open facebook page the server is required to provide the html for the top navigation bar. Whenever you open the news feed an identical html layout for every post needs to be provided with only a few variations in the content of the post.

This forces the server to spend a significnat ammount of time providing redundant data instead of the actual content the user wants.

Due to this issue, the standard jTags are built with the following moto:

The server is ment to provide data not junk !

You can think of "junk" as the results of the repedative nature of servers currently. The server sould not recursivly provide identical information, and should provide the data only when necessary.
